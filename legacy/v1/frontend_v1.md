# Frontend TaskFlow

## Objetivo

O TaskFlow deve seguir visualmente o padrão Boxx/iClips.

## Regras

- Priorizar densidade de informação.
- Evitar grandes áreas vazias.
- Evitar formulários excessivamente espaçados.
- Layout desktop first.
- Painéis laterais estilo iClips.
- Abas horizontais compactas.
- Tabelas densas com alta quantidade de informações.
- Utilizar bordas suaves.
- Utilizar paleta atual do TaskFlow.
- Evitar componentes gigantes.
- Evitar cards com muito padding.

## Referências

- iClips
- Boxx
- ClickUp (somente algumas interações)
- Jira (somente kanban)

## Não fazer

- Visual estilo ERP financeiro.
- Layout mobile-first.
- Telas excessivamente minimalistas.
