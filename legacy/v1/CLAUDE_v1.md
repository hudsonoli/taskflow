# TaskFlow

## Regras gerais

Sempre diagnosticar antes de alterar.

Nunca executar:

- migrate
- seed
- commit

sem autorização explícita.

## Fluxo obrigatório

1. Diagnóstico
2. Plano
3. Aprovação
4. Implementação
5. Testes
6. Relatório final

## Ao finalizar qualquer fase

Informar:

- arquivos criados
- arquivos alterados
- migrations criadas
- rotas criadas
- comandos necessários
- testes realizados

## Banco

Nunca remover migrations antigas.

Nunca criar alterações destrutivas sem autorização explícita.

## Frontend

Seguir padrão visual iClips/Boxx.

Priorizar:

- densidade
- proporção
- usabilidade
- produtividade

## Objetivo do sistema

TaskFlow é um sistema operacional para gestão de:

- tarefas
- projetos
- workflows
- aprovações
- refações
- relatórios operacionais

Não é ERP.
